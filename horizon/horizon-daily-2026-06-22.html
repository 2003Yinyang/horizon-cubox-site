SQLitePython toolingmigrationsdeveloper workflows

English Summary

Among the open-source candidates, this one is the most actionable for day-to-day engineering work. Simon Willison’s release note for `sqlite-utils 4.0rc1` introduces two meaningful capabilities: built-in database migrations and a new `db.atomic()` abstraction for nested transactions built on SQLite savepoints. Those are not cosmetic changes. They move the project closer to being a more complete operational tool for schema evolution and safer write workflows, especially in scripts, local tools, and small data apps that use SQLite as infrastructure. The release candidate also consolidates several earlier breaking changes for version 4, including packaging updates, type-detection defaults, and API behavior changes around tables and views. The main reason to track it now is timing: this is exactly the phase where trying the RC can influence the final stable release. For tool builders, that makes it more valuable than a routine patch release.
